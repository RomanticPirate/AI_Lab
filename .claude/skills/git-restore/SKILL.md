# git-restore — 이전 세이브포인트로 복구

## Overview

이전 세이브포인트로 안전하게 복구한다. 복구 전에 현재 상태를 자동 백업하므로 언제든 다시 돌아올 수 있다. git 용어는 사용자에게 절대 노출하지 않는다.

## 실행 흐름

### 1. repo 확인
```bash
git rev-parse --is-inside-work-tree
```
- 실패 시: "이 폴더는 아직 저장소가 설정되지 않았어요." 안내 후 중단

### 2. 세이브포인트 목록 표시

git-list 스킬과 동일한 형식으로 최근 10개 세이브포인트를 보여준다.

```bash
git log --oneline -10 --format="%h|%ad|%s" --date=format:"%m월 %d일 %H:%M"
```

포맷팅 예시:
```
📋 세이브포인트 목록 (최근 10개)
─────────────────────────────
#1  2월 26일 14:30  로그인 기능 추가
#2  2월 26일 11:00  메인 페이지 레이아웃 완성
#3  2월 25일 16:45  프로젝트 초기 설정
─────────────────────────────
```

### 3. 복구할 세이브포인트 선택

AskUserQuestion으로 "몇 번 세이브포인트로 돌아갈까요?" 질문한다. 선택지는 목록의 각 번호로 제공한다.

### 4. 현재 상태 자동 백업 (안전 복구)
```bash
git add -A
git commit -m "복구 전 자동 백업" --allow-empty
```
- 사용자에게: "현재 상태를 자동 백업했어요. 언제든 돌아올 수 있습니다." 안내

### 5. 선택한 세이브포인트로 복구
```bash
git checkout <선택한 커밋 hash> -- .
```
- `<선택한 커밋 hash>`는 2단계에서 조회한 목록의 해당 번호 commit hash

### 6. 복구 내용 저장
```bash
git add -A
git commit -m "세이브포인트 #N으로 복구"
git push
```

### 7. 완료 안내
```
🔄 세이브포인트 #N으로 복구 완료!
   복구 내용: [해당 세이브포인트 메시지]
   💡 이전 상태는 자동 백업되어 있어 /git-restore로 언제든 돌아갈 수 있어요
```

## 중요 규칙

- checkout, reset, commit hash 같은 git 용어를 안내 메시지에 사용하지 않는다
- 복구 전 반드시 현재 상태를 백업한다 (안전 복구 원칙)
- 모든 안내는 한국어로 한다
- 에러 발생 시 기술적 메시지를 그대로 보여주지 말고, 쉬운 말로 바꿔서 안내한다
