# git-list — 세이브포인트 목록 보기

## Overview

저장된 세이브포인트(커밋) 목록을 사용자 친화적으로 보여준다. git 용어는 사용자에게 절대 노출하지 않는다.

## 실행 흐름

1. **repo 확인**: `git rev-parse --is-inside-work-tree` 실행
   - 실패 시: "이 폴더는 아직 저장소가 설정되지 않았어요." 안내 후 중단

2. **로그 조회**: 아래 명령으로 최근 10개 커밋을 가져온다
   ```bash
   git log --oneline -10 --format="%h|%ad|%s" --date=format:"%m월 %d일 %H:%M"
   ```

3. **포맷팅**: 결과를 아래 형식으로 출력한다. 번호는 최신순 #1부터 매긴다.

```
📋 세이브포인트 목록 (최근 10개)
─────────────────────────────
#1  2월 26일 14:30  로그인 기능 추가
#2  2월 26일 11:00  메인 페이지 레이아웃 완성
#3  2월 25일 16:45  프로젝트 초기 설정
─────────────────────────────
💡 이전 상태로 돌아가려면 /git-restore를 사용하세요
```

4. 커밋이 없으면: "아직 저장된 세이브포인트가 없어요. /git-save로 첫 세이브포인트를 만들어보세요!" 안내

## 중요 규칙

- commit hash는 내부 처리용으로만 사용하고 사용자에게 보여주지 않는다
- git log, commit, hash 같은 기술 용어를 안내 메시지에 사용하지 않는다
- 모든 안내는 한국어로 한다
