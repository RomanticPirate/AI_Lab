# git-load — 최신 코드 불러오기

## Overview

GitHub에서 최신 코드를 불러온다. 로컬에 저장하지 않은 변경사항이 있으면 자동으로 먼저 저장한다. git 용어는 사용자에게 절대 노출하지 않는다.

## 실행 흐름

### 1. repo 확인
```bash
git rev-parse --is-inside-work-tree
```
- 실패 시: "이 폴더는 아직 저장소가 설정되지 않았어요." 안내 후 중단

### 2. 로컬 변경사항 확인 (예방)
```bash
git status --porcelain
```
- 변경사항 있음 → "저장하지 않은 작업이 있어요. 먼저 저장할게요!" 안내 후 git-save 로직 실행
- 변경사항 없음 → 3단계로

### 3. 최신 코드 불러오기
```bash
git pull
```
- 성공 → 4단계로
- 충돌 발생 → **충돌 해결 플로우**로 이동

### 4. 완료 안내

변경된 내용이 있는 경우:
```
✅ 최신 코드를 불러왔습니다!
   업데이트된 파일: [변경된 파일 목록 요약]
```

이미 최신인 경우:
```
✅ 이미 최신 상태예요! 새로 불러올 내용이 없습니다.
```

## 충돌 해결 플로우

자동 머지를 시도하고, 실패하면 사용자에게 쉬운 한국어로 선택지를 제안한다.

### 자동 해결 실패 시 — AskUserQuestion으로 선택지 제안

아래 내용을 사용자에게 보여준다:

```
⚠️ 불러오려는 내용과 내 컴퓨터의 내용이 겹치는 부분이 있어요!
```

선택지:
1. **[추천] GitHub에 있는 최신 내용으로 가져오기** — `git checkout --theirs .` 후 add/commit/push
2. **내 컴퓨터에 있는 내용을 유지하기** — `git checkout --ours .` 후 add/commit/push
3. **Claude가 두 내용을 합쳐보기** — 충돌 파일을 읽고 수동으로 병합 시도

선택 후 결과를 안내한다.

## 중요 규칙

- pull, merge, fetch, conflict 같은 git 용어를 안내 메시지에 사용하지 않는다
- 모든 안내는 한국어로 한다
- 에러 발생 시 기술적 메시지를 그대로 보여주지 말고, 쉬운 말로 바꿔서 안내한다
