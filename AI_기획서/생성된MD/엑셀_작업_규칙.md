---
name: 엑셀 작업 규칙
description: 엑셀 생성 시 빈 셀 서식 확인, 명시적 RGB 사용, 시각적 결과 기준 판단
type: feedback
---

1. 빈 셀의 서식(fill, font 등)도 반드시 확인하고 재현할 것.

**Why:** 기획서 포맷에서 빈 셀에도 배경색이 칠해져 있어 시각적 구조를 형성하는데, 이걸 무시하면 포맷이 완전히 달라진다.

**How to apply:** 엑셀 파일 분석 시 모든 셀(빈 셀 포함)의 fill, font, alignment 속성을 체크.

2. 테마 색상(theme=N, tint=X) 대신 명시적 RGB 색상을 사용할 것.

**Why:** 테마 색상은 워크북의 theme XML에 따라 렌더링이 달라진다. 원본에서 theme=1/tint=0.15로 보이는 색이 새 워크북에서는 다른 색으로 표시될 수 있다.

**How to apply:** 원본의 테마 색상이 실제로 어떤 RGB인지 확인한 후, 명시적 RGB(예: FF262626)로 적용.

3. 셀 단위 raw 데이터만 보지 말고, 실제 엑셀에서 보이는 시각적 결과를 기준으로 판단할 것.

**Why:** 큰 제목이 B+C 2칸에만 검정 fill이 있어도, 시각적으로는 가로로 길게 검정 띠가 보여야 한다. 데이터 분석 결과와 사용자가 보는 화면은 다를 수 있다.

**How to apply:** 헤더/구분선 등 시각적 요소는 충분한 범위에 fill을 적용. column style로 기본 적용되는 서식도 확인.

4. 배경색 fill 범위는 최소 DC열(108열)×1000행까지 적용할 것.

**Why:** 엑셀 화면에서 fill이 적용된 영역 밖이 보이면 잘린 것처럼 보인다. 내용이 없는 하단 영역도 반드시 색칠.

**How to apply:** MAX_COL=108, MAX_ROW=1000을 기본값으로 사용.

5. 기획서 강조 표현 규칙 (시트로 넣지 않고 내용 작성 시 적용):
   - 중요한 내용 → bold
   - 더 중요한 내용 → 주황색(#FF8C00) 텍스트
   - 가장 중요한 내용 → bold + 주황색 텍스트

6. 첫번째 시트 이름은 해당 문서의 제목으로 설정할 것. "디자인 구성 정보" 같은 작성 룰 시트는 넣지 않음.

7. 표(Table) 작성 규칙:
   - 각 필드는 텍스트 길이에 맞게 셀 병합(merge)으로 가로 폭을 확보할 것
   - 모든 표 셀에 회색(#808080) thin 테두리 적용
   - 헤더 행은 어두운 회색(#404040) 배경색 적용

**Why:** 기본 열 너비(2.625)가 좁아서 병합 없이는 텍스트가 잘린다. 테두리와 헤더 배경이 있어야 표로 인식된다.

**How to apply:** openpyxl merge_cells + Border(thin, #808080) + 헤더에 PatternFill(#404040).
